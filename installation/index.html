Installation Instructions

DYCOVE is compatible with Python 3.11+. There are technically only two general dependencies, which are numpy and netCDF4. However, there are additional dependencies depending on the underlying numerical model to be used, and if the ModelPlotter will be used.

Install ANUGA

For DYCOVE with ANUGA, anuga must be installed. ANUGA may have additional dependencies depending on the application, such as mpi4py for running in parallel. The easiest way to install ANUGA is to create a new conda environment and install via conda-forge (from the ANUGA documentation page):

conda install -c conda-forge anuga mpi4py

Install Delft3D FM

For DYCOVE with Delft3D FM, the software itself will of course need to be licensed and installed locally. From a Python standpoint, it is also recommended to create a new environment to install DYCOVE and its dependency bmi, which provides the interface between Python and the Delft3D FM software. From a dedicated environment, the easiest way to install bmi is from source:

git clone https://github.com/openearth/bmi-python.git
cd bmi-python
pip install -e .

Windows users may also need to install pypiwin32, which can be done via pip:

pip install pypiwin32

Install Plotting Libraries

DYCOVE’s ModelPlotter is a convenient way to plot outputs. The dependencies are matplotlib, scipy, tqdm, and imageio. Of course, users will want to have some of these libraries installed whether or not they plan to use ModelPlotter.

Installing DYCOVE

DYCOVE itself can be installed from source like any other package:

git clone https://github.com/Ecomomo-lab/dycove-model.git
cd dycove-model
pip install -e .
